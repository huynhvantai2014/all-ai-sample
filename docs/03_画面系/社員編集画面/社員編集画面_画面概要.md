# 画面概要

---

**画面名**：社員編集画面
**画面ID**：SRC0006
**画面物理名**：employee_edit

### 目的・業務価値
既存社員の情報を安全且つ効率的に編集・更新することで、人材データベースの正確性を維持し、組織変更や職務変更に迅速に対応する。データ整合性を保ちながら柔軟な情報管理を実現する。

### 主な機能
- 既存社員データの自動入力・表示
- 社員基本情報編集（氏名、連絡先、部門、職位等）
- スキル情報の追加・削除・編集
- 雇用ステータス変更
- リアルタイム入力バリデーション
- データ重複チェック（メール、社員ID）
- 更新確認ダイアログ
- 更新成功・失敗通知
- React Hook Form による統合フォーム管理

### 業務フロー
1. 社員詳細画面から編集ボタンクリック
2. 社員ID検証・データ取得
3. 既存データをフォームに自動入力
4. 管理者が必要項目を編集
5. バリデーション実行（必須項目・形式チェック）
6. 重複チェック（メール・社員ID）
7. 確認ダイアログで更新確認
8. データベース更新実行
9. 成功時：詳細画面に戻る、失敗時：エラー表示

### 利用者
- HR管理者（全社員情報編集権限）
- 管理者（全社員情報編集権限）
- マネージャー（部門内社員情報編集権限）

### 関連・遷移画面
- **遷移元**：社員詳細画面（/employees/[employee_id]）- 編集ボタン
- **遷移先**：社員詳細画面（/employees/[employee_id]）- 更新成功時
- **遷移先**：社員一覧・検索画面（/employees）- キャンセル・戻るボタン

### 代表的な利用シナリオ
1. **部門異動**：社員の部門変更 → 部門・職位更新 → 確認・保存
2. **連絡先更新**：社員からの連絡先変更依頼 → メール・電話番号更新
3. **スキル情報更新**：新たな資格取得・技能習得時のスキル追加
4. **ステータス変更**：退職・休職・復職時のステータス更新
5. **職位昇進**：昇進・昇格時の職位・役職更新

### 特記事項
- 重要データ変更のため確認ダイアログ必須
- 変更履歴のログ記録（監査対応）
- 同時編集防止機能（楽観的ロック）
- 必須項目の明確な表示
- モバイル・タブレット最適化：縦スクロール対応、フォーム項目の適切な間隔、タッチ操作対応
- バリデーションエラーの項目別表示
