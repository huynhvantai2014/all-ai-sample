# 画面概要

---

**画面名**：ホーム画面
**画面ID**：SRC0001
**画面物理名**：home

### 目的・業務価値
システムのランディングページとして、未認証ユーザーへのシステム紹介および認証済みユーザーの適切な画面への誘導を行う。新規ユーザーにとってのシステム理解の入り口となり、既存ユーザーには円滑なシステム利用を提供する。

### 主な機能
- システム概要・主要機能の紹介表示
- 認証状態の自動判定
- 認証済みユーザーのダッシュボードへの自動リダイレクト
- 未認証ユーザーのログインページへの誘導
- レスポンシブデザインによるマルチデバイス対応
- 人材管理・リソースダッシュボード・効率化機能の紹介

### 業務フロー
1. ユーザーがシステムにアクセス
2. 認証状態確認（localStorageのトークンチェック）
3. 認証済み：ダッシュボードへ自動リダイレクト
4. 未認証：ウェルカム画面表示
5. システム機能紹介の閲覧
6. ログインボタンクリックでログイン画面へ遷移

### 利用者
- 初回アクセスユーザー（システム理解）
- 未認証ユーザー（システム紹介・ログイン誘導）
- 認証済みユーザー（ダッシュボードへの通過点）

### 関連・遷移画面
- **遷移先**：ログイン画面（/login）- ログインボタンクリック時
- **自動遷移**：リソースダッシュボード（/dashboard）- 認証済みユーザー

### 代表的な利用シナリオ
1. **新規ユーザー**：システムURLにアクセス → 機能紹介を閲覧 → ログインボタンでログイン画面へ
2. **既存ユーザー（ダイレクトアクセス）**：システムURLにアクセス → 自動でダッシュボードにリダイレクト
3. **ログアウト後ユーザー**：ログアウト後にホームに戻る → システム紹介再確認 → 再ログイン

### 特記事項
- 認証状態による画面制御が重要（useEffectでのlocalStorage確認）
- SEO対応としてシステム概要の適切な記述
- モバイル・タブレット最適化：カード型レイアウト、タッチ操作に適したボタンサイズ、縦スクロール対応
- Material UIのレスポンシブGrid systemによる柔軟なレイアウト
