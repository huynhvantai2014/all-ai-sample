# ホーム画面_画面概要# 画面概要



## 業務目的---

システムのエントリーポイントとして、認証状態に応じて適切な画面へ誘導する。未認証時はウェルカム画面、認証済みの場合はダッシュボードやリソース一覧画面へ遷移。

**画面名**：ホーム画面

## 主な機能**画面ID**：SRC0001

- 認証状態判定**画面物理名**：home

- 画面遷移（ログイン画面、リソース一覧画面）

- システム概要表示### 目的・業務価値

システムのランディングページとして、未認証ユーザーへのシステム紹介および認証済みユーザーの適切な画面への誘導を行う。新規ユーザーにとってのシステム理解の入り口となり、既存ユーザーには円滑なシステム利用を提供する。

## 主業務フロー

1. ユーザーがアクセス### 主な機能

2. 認証状態を判定- システム概要・主要機能の紹介表示

3. 未認証ならログイン画面へ、認証済みならリソース一覧画面へ遷移- 認証状態の自動判定

- 認証済みユーザーのダッシュボードへの自動リダイレクト

## 利用者- 未認証ユーザーのログインページへの誘導

- 全ユーザー- レスポンシブデザインによるマルチデバイス対応

- 人材管理・リソースダッシュボード・効率化機能の紹介

### 特記事項
- 認証状態による画面制御が重要（useEffectでのlocalStorage確認）
- SEO対応としてシステム概要の適切な記述
- **モバイル・タブレット最適化（UI/UX）**：
  - レスポンシブカード型レイアウト（Grid system使用）
  - タッチ操作対応：ボタンサイズ44px以上、適切な間隔確保
  - ワンタッチ操作重視：大型ログインボタン、機能紹介カードから直接アクション
  - 情報密度最適化：重要情報を上部配置、スクロール最小化
  - 画面遷移最小化：認証済みユーザーの自動リダイレクト
  - 縦スクロール対応、横スクロール不要の設計
- Material UIのレスポンシブGrid systemによる柔軟なレイアウト
