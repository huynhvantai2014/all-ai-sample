# ホーム画面_画面概要

## 目的・業務価値
システムのエントリーポイントとして、認証状態に応じて適切な画面へ誘導する。未認証ユーザーには人材管理システムの概要紹介とログイン誘導を行い、認証済みユーザーはリソース一覧画面へ自動遷移する。新規ユーザーのシステム理解促進と既存ユーザーの円滑な利用開始を実現する。

## 主な機能
- 認証状態の自動判定とルーティング制御
- システム概要・主要機能の紹介表示
- 認証済みユーザーのダッシュボードへの自動リダイレクト  
- 未認証ユーザーのログインページへの誘導
- レスポンシブデザインによるマルチデバイス対応
- 通知表示、ヘルプ機能、言語切替、利用者プロフィール管理

## 業務フロー
1. ユーザーがシステムにアクセス
2. 認証状態を自動判定（localStorage.getToken()）
3. 未認証：ウェルカム画面表示とシステム機能紹介
4. 認証済み：リソース一覧画面へ自動遷移
5. 未認証ユーザー：ログインボタンクリックでログイン画面へ遷移

## 利用者
- 未認証ユーザー（システム紹介・ログイン誘導対象）
- 認証済みユーザー（自動リダイレクト対象）
- 管理者、一般社員、ゲスト

## 画面遷移
| 遷移元画面 | 遷移方法 | 条件 | 遷移先画面 |
|------------|--------|------|------------|
| - | 直接アクセス | 未認証状態 | ホーム画面（現在画面） |
| - | 直接アクセス | 認証済み状態 | リソース一覧画面 |
| ホーム画面 | ログインボタンクリック | 未認証状態 | ログイン画面 |
| 各種画面 | ログアウト | - | ホーム画面 |
