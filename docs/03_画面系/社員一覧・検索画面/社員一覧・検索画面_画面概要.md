# 画面概要

---

**画面名**：社員一覧・検索画面
**画面ID**：SRC0004
**画面物理名**：employees

### 目的・業務価値
全社員の情報を一覧表示し、効率的な検索・フィルタリング機能を提供することで、HR管理者が迅速な人材把握と意思決定を行えるようにする。人材配分・管理業務の中核となる画面として、データドリブンなHR業務を支援する。

### 主な機能
- 全社員一覧表示（社員ID、氏名、メール、部門、職位、ステータス、入社日）
- キーワード検索（氏名、メール、社員ID）
- 部門別フィルタリング
- ステータス別フィルタリング（在職/退職/休暇中）
- ページネーション対応
- 検索結果件数表示
- 社員詳細画面への遷移
- 社員編集画面への遷移
- CSV出力機能（実装予定）
- 新規社員追加画面への遷移

### 業務フロー
1. ダッシュボードまたは直接アクセスで画面表示
2. 認証チェック・権限確認
3. 全社員データの取得・表示
4. 検索キーワード入力またはフィルタ選択
5. 条件に応じた検索結果表示
6. 対象社員の詳細表示または編集
7. 必要に応じてCSV出力

### 利用者
- HR管理者（全社員管理）
- 管理者（全社員アクセス）
- マネージャー（部門内社員管理）

### 関連・遷移画面
- **遷移元**：リソースダッシュボード（/dashboard）- 人材一覧ボタン
- **遷移先**：社員詳細画面（/employees/[employee_id]）- 詳細ボタン
- **遷移先**：社員編集画面（/employees/[employee_id]/edit）- 編集ボタン
- **遷移先**：新規社員追加画面（/employees/add）- 新規登録ボタン

### 代表的な利用シナリオ
1. **人材検索**：プロジェクト配分前の適任者検索 → キーワード・部門で絞り込み
2. **状況確認**：部門別の在籍状況確認 → 部門フィルタで一覧表示
3. **情報更新**：社員情報の確認・修正 → 検索→詳細→編集
4. **レポート作成**：人事レポート作成 → フィルタ→CSV出力
5. **新規登録**：新入社員の情報登録 → 新規追加画面へ遷移

### 特記事項
- JWT認証による閲覧権限制御
- 大量データに対応したページネーション実装
- リアルタイム検索・フィルタリング
- モバイル・タブレット最適化：レスポンシブテーブル、横スクロール対応、タッチ操作最適化
- 検索・フィルタボタンの大型化、画面遷移最小化のための多機能ボタン配置
- 情報密度最適化（重要情報優先表示、詳細は展開式）
