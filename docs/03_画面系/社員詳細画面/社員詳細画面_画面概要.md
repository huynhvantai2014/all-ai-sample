# 画面概要

---

**画面名**：社員詳細画面
**画面ID**：SRC0005  
**画面物理名**：employee_detail

### 目的・業務価値
個別社員の詳細情報を包括的に表示し、HR管理者が社員の基本情報、職務情報、スキル情報を正確に把握できるようにする。社員情報の確認・検証および編集への導線として重要な役割を担う。

### 主な機能
- 社員基本情報詳細表示（社員ID、氏名、連絡先、部門、職位）
- 雇用ステータス表示（在職/退職/休暇中）
- スキル情報の配列表示
- 入社日・システム登録日時の表示
- 編集画面への遷移機能
- 一覧画面への戻り機能
- 認証による閲覧権限制御

### 業務フロー
1. 社員一覧画面から対象社員の詳細ボタンクリック
2. 社員ID検証・存在確認
3. データベースから社員詳細情報取得
4. 基本情報・職務情報・スキル情報の表示
5. 必要に応じて編集ボタンで編集画面へ遷移
6. 戻るボタンで一覧画面に戻る

### 利用者
- HR管理者（社員情報確認・管理）
- 管理者（全社員情報アクセス）
- マネージャー（部門内社員情報確認）

### 関連・遷移画面
- **遷移元**：社員一覧・検索画面（/employees）- 詳細ボタン
- **遷移先**：社員編集画面（/employees/[employee_id]/edit）- 編集ボタン
- **遷移先**：社員一覧・検索画面（/employees）- 戻るボタン

### 代表的な利用シナリオ
1. **情報確認**：一覧から気になる社員を選択 → 詳細情報確認 → 一覧に戻る
2. **編集前確認**：編集対象社員の現在情報を確認 → 編集画面で修正
3. **スキル確認**：プロジェクト配属前に社員のスキルセット確認
4. **問い合わせ対応**：社員からの問い合わせ時に詳細情報確認

### 特記事項
- 存在しない社員IDアクセス時の適切なエラーハンドリング
- JWT認証による閲覧権限制御
- 個人情報保護のためのアクセスログ記録（将来実装）
- モバイル・タブレット最適化：縦型レイアウト、読みやすいフォントサイズ、タッチ操作対応
- ステータスに応じた視覚的表示（Chipコンポーネント使用）
