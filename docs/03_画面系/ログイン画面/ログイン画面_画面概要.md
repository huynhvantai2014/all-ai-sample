# ログイン画面_画面概要

## 1. 業務目的
システム利用者が個人アカウントで安全にログインし、権限に応じた機能へアクセスする。セキュリティ確保・操作履歴管理。

## 2. 主な機能
- ユーザー名・パスワード入力
- OTP認証（任意）
- 権限判定（管理者/社員）
- ログイン・ログアウト履歴表示
- セッション管理・タイムアウト
- （テスト環境限定）ユーザー種別選択機能（管理者/社員/ゲスト等を選択してテスト可能）

## 3. 主業務フロー
1. ユーザーがログイン画面を開く
2. ユーザー名・パスワード入力
3. OTP認証（必要時）
4. 権限判定・画面遷移
5. ログイン履歴保存

## 4. 利用者
- 管理者
- 社員

## 5. 他画面との関係
- ダッシュボード画面（ログイン後遷移）
- パスワード再設定画面（リンク）

## 6. 代表的な利用ケース
- 毎日の出社時ログイン
- 管理者による権限確認

## 7. 特記事項
- モバイル・タブレット最適化（UI/UX）：
  - レスポンシブレイアウト
  - タッチ最適化（大きめボタン・入力欄）
  - 情報量最小化、優先度高い項目のみ表示
  - ワンタッチ操作、画面遷移最小化
  - 多機能ボタン配置、業務迅速化
- セキュリティ強化（OTP/パスワードポリシー）
- 情報不足時は「情報不足」と記載
