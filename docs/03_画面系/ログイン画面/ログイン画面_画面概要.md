# 画面概要

---

**画面名**：ログイン画面
**画面ID**：SRC0001
**画面物理名**：login

### 目的・業務価値
POS端末へのユーザー認証を行い、各取引を担当者・勤務シフトに紐付けて管理・監査を可能にする。

### 主な機能
- ユーザー名・パスワードによる認証
- 現在の店舗・端末情報表示
- 「Remember device」オプション（固定端末用）
- ログイン後、ユーザー名・ロール・シフト表示
- セッションタイムアウト・再認証/管理者解除
- ログイン/ログアウト監査（タイムスタンプ、端末ID、IP）

### 業務フロー
1. ユーザーが端末でログイン画面を開く
2. ユーザー名・パスワードを入力
3. 認証成功時、店舗・端末・ユーザー情報を表示
4. セッション管理（タイムアウト時は再認証）
5. ログイン/ログアウト履歴を記録

### 利用者
- POS担当者（店員、レジ係）

### 関連・遷移画面
- ダッシュボード画面（ログイン後遷移）
- 仮販売作成画面

### 代表的な利用シナリオ
- POS担当者が勤務開始時に端末へログイン
- シフト交代時に再ログイン

### 特記事項
- 端末認証・セキュリティ強化
- 監査ログ保存

### 情報不足
なし
