# ログイン画面_画面概要# 画面概要



## 業務目的---

ユーザー認証を行い、システムへの安全なアクセスを提供する。

**画面名**：ログイン画面

## 主な機能**画面ID**：SRC0002

- ユーザーID・パスワード入力**画面物理名**：login

- JWT認証

- ログイン成功時のリソース一覧画面への遷移### 目的・業務価値

HR管理システムへの安全なアクセスポイントとして、JWT認証による役割ベースアクセス制御を提供し、セキュリティを確保しながら適切な権限でシステム利用を可能にする。セッション管理により効率的で安全な業務環境を実現する。

## 主業務フロー

1. ユーザーがID・パスワードを入力### 主な機能

2. 認証処理を実行- ユーザーID・パスワードによるログイン認証

3. 成功時はリソース一覧画面へ遷移、失敗時はエラーメッセージ表示- JWT トークンベースの認証システム

- 役割別権限管理（admin/manager/general）

## 利用者- ログイン成功時のダッシュボード自動リダイレクト

- 全ユーザー- ローカルストレージでのトークン・役割情報保存

- セッションタイムアウト機能

## 他画面との関係- テスト環境用ユーザー種別選択機能

- ホーム画面- 認証エラー時の適切なエラーメッセージ表示

- リソース一覧画面

### 業務フロー

## 代表的な利用ケース1. ユーザーがログイン画面アクセス

- システム初回ログイン2. ユーザーID・パスワード入力

- セッション切れ時の再認証3. テスト環境の場合：ユーザー種別選択

4. ログインボタンクリック
ユーザー認証を行い、システムへの安全なアクセスを提供する。パスワードポリシー、二段階認証、エラー表示、ヘルプ、通知、履歴、権限管理、レスポンシブ対応。
## 特記事項5. 認証処理・JWT トークン生成

## 代表的な利用ケース
- システム初回ログイン
- セッション切れ時の再認証
- 二段階認証
- パスワード再設定
- エラー・ヘルプ・通知・履歴・権限の確認
- **遷移先**：リソースダッシュボード（/dashboard）- ログイン成功時
- **遷移元**：各種画面 - 認証エラー・セッションタイムアウト時
## 特記事項
- パスワードポリシー
- セキュリティ強化（JWT）
- 二段階認証
- エラー表示
- ヘルプ・通知・履歴・権限管理
- レスポンシブデザイン
- アクセシビリティ配慮
1. **日常ログイン**：出社時のシステムアクセス → ID/パスワード入力 → ダッシュボードで業務開始
2. **権限変更後**：昇進・部署異動後の初回ログイン → 新しい権限でのアクセス確認
3. **テスト環境利用**：開発者がテスト実行 → ユーザー種別選択 → 各権限での動作確認
4. **セッション復帰**：タイムアウト後の再ログイン → 前回作業の継続

### 特記事項
- JWT トークンセキュリティ（有効期限・署名検証）
- パスワードの適切なハッシュ化（bcryptjs使用）
- **モバイル・タブレット最適化（UI/UX）**：
  - シンプル・クリーンなフォームレイアウト
  - タッチ操作対応：入力フィールド・ボタンサイズ44px以上
  - ワンタッチログイン：記憶済みアカウントの自動入力サポート
  - エラーメッセージの分かりやすい表示
  - 画面遷移最小化：成功時の即座リダイレクト
  - レスポンシブデザイン：全デバイス対応
- テスト環境での動的ユーザー種別切り替え機能
