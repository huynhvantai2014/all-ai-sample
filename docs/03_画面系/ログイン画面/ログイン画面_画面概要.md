# ログイン画面_画面概要

## 目的・業務価値
HR管理システムへの安全なアクセスポイントとして、JWT認証による役割ベースアクセス制御を提供し、セキュリティを確保しながら適切な権限でシステム利用を可能にする。テスト環境での複数ユーザー種別切り替え機能により、開発・検証効率を向上させる。

## 主な機能
- ユーザーID・パスワードによるログイン認証
- JWT トークンベースの認証システム
- 役割別権限管理（admin/manager/general）
- ログイン成功時のダッシュボード自動リダイレクト
- ローカルストレージでのトークン・役割情報保存
- セッションタイムアウト機能
- テスト環境用ユーザー種別選択機能
- 認証エラー時の適切なエラーメッセージ表示

## 業務フロー
1. ユーザーがログイン画面アクセス
2. ユーザーID・パスワード入力
3. テスト環境の場合：ユーザー種別選択
4. ログインボタンクリック
5. 認証処理・JWT トークン生成
6. 成功時：トークンをlocalStorageに保存後、リソース一覧画面へ遷移
7. 失敗時：エラーメッセージ表示

## 利用者
- HR管理者（全機能アクセス）
- マネージャー（部門管理機能アクセス）
- 一般社員（参照権限）
- ゲストユーザー（限定権限）

## 画面遷移
| 遷移元画面 | 遷移方法 | 条件 | 遷移先画面 |
|------------|--------|------|------------|
| ホーム画面 | ログインボタンクリック | 未認証状態 | ログイン画面（現在画面） |
| 各種画面 | 認証エラー | トークン無効・期限切れ | ログイン画面（現在画面） |
| ログイン画面 | ログイン成功 | 認証成功 | リソース一覧画面 |
| ログイン画面 | キャンセル | - | ホーム画面 |
