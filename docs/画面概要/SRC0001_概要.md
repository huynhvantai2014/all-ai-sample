# 画面概要

---

**画面名**：ログイン画面
**画面ID**：SRC0001
**画面物理名**：login_screen

### 目的・業務価値
POS端末へのユーザー認証を行い、各取引を担当者・勤務シフトに紐付けて管理・監査できるようにする。セキュリティ強化・業務トレーサビリティ確保、内部統制・法令遵守のため必須。

### 主な機能
- ユーザー名・パスワードによる認証
- 店舗・端末情報の表示
- 「Remember device」オプション（固定端末用）
- ログイン後、ユーザー名・権限・シフト情報表示
- セッションタイムアウト・再認証/管理者解除
- ログイン・ログアウト監査記録（タイムスタンプ、端末ID、IP）
- （開発環境のみ）テスト用アカウント一覧表示・選択、認証情報自動入力

### 業務フロー
1. POS端末起動時、ログイン画面表示
2. ユーザーが認証情報（ユーザー名・パスワード）入力
3. （開発環境）テストアカウント一覧から選択→自動入力
4. 認証成功時、ユーザー情報・シフト情報を画面上に表示
5. 業務開始。タイムアウト時は再認証または管理者解除
6. 業務終了時にログアウトし、監査ログ記録

### 利用者
- POS担当者（レジ係）
- 管理者（監査・設定）
- （開発環境）開発者・テスター

### 関連・遷移画面
- 仮販売作成画面（SRC0002）
- 仮販売一覧画面（SRC0003）
- レジ販売画面（SRC0004）

### 代表的な利用シナリオ
- レジ係が勤務開始時にPOSへログインし、取引を開始
- シフト交代時に再ログインして担当者切替
- セッションタイムアウト後、再認証して業務継続
- 管理者が監査目的でログイン履歴を確認
- （開発環境）テストアカウントを選択して素早くログイン・動作検証

### 特記事項
- 監査ログは法令遵守・内部統制のため必須
- 固定端末は「Remember device」利用推奨
- セキュリティ強化のため、パスワードポリシー・多要素認証拡張可能
- 端末ID・IP記録は不正利用防止・追跡のため重要
- 開発環境ではテストアカウント選択・自動入力機能を有効化
