# 共通レイアウト画面概要（IPA標準フォーマット）

---

**画面名**：共通レイアウト（ヘッダー・フッター）
**画面ID**：COMMON_LAYOUT
**画面物理名**：common_layout

### 目的・業務価値
全POS画面で一貫した操作性・情報表示を提供し、ユーザー体験と業務効率を向上させる。店舗・端末・IP等の環境情報を明示し、業務の正確性・セキュリティ・トレーサビリティを確保。

### 主な機能
- ヘッダー：
    - 画面タイトル、ロゴ表示
    - 店舗名、端末ID、IPアドレス等の環境情報（例：店舗: 新宿店 | 端末ID: POS-001 | IP: 192.168.1.10）
    - ユーザー名、ログアウトボタン
- フッター：
    - システム名、バージョン、コピーライト
    - サポート・問い合わせ情報
- レスポンシブ対応
- （創造）業務切替・通知領域

### 利用者
- 全POS利用者（レジ担当者、管理者、IT管理者等）

### 関連・遷移画面
- ログイン画面（SRC0001）
- レジ販売画面（SRC0004）
- その他全POS画面

### 代表的な利用シナリオ
- ログイン後、ヘッダーで店舗・端末情報を確認
- 業務中、フッターでシステムバージョンやサポート情報を参照

### 特記事項
- 店舗・端末・IP情報は全画面で共通表示
- レイアウトは画面設計のベースとなる
- 画面ごとに追加情報領域を拡張可能
