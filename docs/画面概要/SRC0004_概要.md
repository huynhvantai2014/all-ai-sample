# 画面概要（IPA標準フォーマット）

---

**画面名**：レジ販売画面
**画面ID**：SRC0004
**画面物理名**：checkout_screen

### 目的・業務価値
POSレジ担当者が商品販売・決済を迅速かつ正確に処理するための中心画面。バーコード・JANコード・SKU対応により業務効率化、顧客満足度向上、売上・在庫管理の精度向上に寄与。

### 主な機能
- 商品バーコード・JANコード・SKUによる商品追加・削除
- 数量・割引・プロモーション・バウチャー適用
- 税率（消費税10%）自動計算・表示
- 合計金額・税額・明細表示
- 商品ごとの数量調整・割引適用
- 決済方法選択（現金・クレジット・IC・QR・分割払い）
- 領収書・電子レシート発行
- （創造）商品画像表示・タッチ選択（UI向上）
- **商品リスト（カート）表示：**
    - 商品名、SKU/JAN、画像、数量、単価、割引、合計金額
    - 各行に「編集」「削除」ボタン
    - 合計金額・税額の自動更新
    - 空の場合は「カートは空です」表示

### 業務フロー
1. レジ担当者が商品をスキャンまたはSKU/JANコード入力
2. 商品リスト（カート）に追加・表示
3. 数量・割引・バウチャー等を調整
4. 合計金額・税額を確認
5. 決済方法を選択し、決済処理
6. 領収書・電子レシートを発行

### 利用者
- レジ担当者（POSオペレーター）
- 顧客（決済・領収書受領）
- 管理者（売上・在庫管理）

### 関連・遷移画面
- ログイン画面（SRC0001）
- 仮販売一覧画面（SRC0003）
- 決済選択画面（SRC0005）

### 代表的な利用シナリオ
- レジ担当者が商品をスキャンし、割引・バウチャーを適用して決済
- 顧客が複数商品を購入し、領収書・電子レシートを受領
- 管理者が売上・在庫データを確認

### 特記事項
- 商品画像表示・タッチ選択はUI/UX向上に有効
- 税率・割引・バウチャーは法令・社内規定に準拠
- 領収書・電子レシートは内部統制・監査対応
- 商品リスト（カート）表示は必須。編集・削除・数量調整が可能。
