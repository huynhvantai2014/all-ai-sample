## 役割
LLM（大規模言語モデル）を活用し、画面一覧（docs/03_画面系/画面一覧.md）に記載されたすべての画面について、設計ドキュメント（画面概要・画面定義書）から高品質なWebフロントエンドソースコード（HTML5・Tailwind CSS・JavaScript ES6+）を一括自動生成する。

【重要】本手順で使用する技術は「HTML5」「Tailwind CSS」「JavaScript（ES6+）」のみとし、React・Next.js等のフレームワークやライブラリは一切使用しないこと。

## インプット
- 必ず最初に docs/00_intructions/00_system-infrastructure.md（インフラ仕様・技術スタック）を熟読すること
- docs/03_画面系/<画面論理名>/<画面論理名>_画面概要.md（画面概要）
- docs/03_画面系/<画面論理名>/<画面論理名>_画面定義書.md（画面定義書）
- 命名規則: docs/00_intructions/00_naming_and_structure_reference.md

## 実行手順
1. 画面概要・画面定義書・インフラ仕様をすべて読み、画面の業務目的・主要機能・UI要素・イベント・遷移・権限・バリデーション・レイアウト・レスポンシブ要件を把握する。
2. 画面概要の「画面イメージ」や業務フローをもとに、HTML構造・UI配置・主要ボタン・入力欄・リスト等を設計する。生成されたHTMLは「画面イメージ」のレイアウト・配置・要素と完全に一致させること。
3. 画面定義書の「項目定義」「イベント定義」「アクション一覧」から、必要なinput, button, list, modal, event handler, validation, 権限制御等を抽出し、HTML・JSに落とし込む。
4. Tailwind CSSでデザイン・レイアウト・レスポンシブ対応を実装。
5. JavaScript（ES6+、クラス/モジュール可）でロジック（検索、カート、計算、イベント処理等）を実装。
6. バックエンドAPIやDB連携は原則不要。必要な場合は「情報不足」コメントやダミーデータで代用。
7. 命名規則・フォルダ構成を厳守し、src/<画面物理名>/にファイルを配置（フォルダ名は英語）。
8. UI/UX観点（モバイル最適化、直感的操作、アクセシビリティ等）も必ず考慮。

## モバイル最適化 - コンポーネント間スペーシング
**スマートフォン向けUI設計における各コンポーネント間の適切な間隔指定:**

### 基本原則
- **最小タップターゲットサイズ**: 44px × 44px（Apple HIG）または 48dp（Material Design）
- **コンポーネント間マージン**: 最低12px、推奨16px-24px
- **セクション間スペーシング**: 24px-32px（明確な区切りが必要）

### Tailwind CSSクラス指定
```css
/* コンポーネント間基本スペーシング */
.space-y-4     /* 16px縦方向間隔（基本） */
.space-y-6     /* 24px縦方向間隔（セクション間） */
.space-x-4     /* 16px横方向間隔（ボタン並び） */

/* パディング指定 */
.p-3           /* 12px全方向パディング（コンパクト） */
.p-4           /* 16px全方向パディング（基本） */
.py-3 .px-4    /* 縦12px・横16px（ボタン内余白） */

/* マージン指定 */
.mb-4          /* 下マージン16px（基本要素間） */
.mb-6          /* 下マージン24px（セクション間） */
.mt-8          /* 上マージン32px（大きなセクション区切り） */

/* ガップ指定（Flexbox/Grid） */
.gap-4         /* 16px基本ガップ */
.gap-6         /* 24px大きめガップ */
```

### 用途別スペーシング推奨値
1. **フォーム要素間**: `space-y-4`（16px）
2. **ボタン群**: `gap-4`（16px）または`space-x-4`
3. **カード・セクション間**: `space-y-6`（24px）
4. **ヘッダー↔本文**: `mb-6`（24px）
5. **ページ全体パディング**: `p-4`（16px）
6. **リスト項目間**: `space-y-3`（12px）

### レスポンシブ調整
```css
/* 画面サイズ別調整例 */
.space-y-4 .md:space-y-6    /* 小画面16px、中画面以上24px */
.p-4 .md:p-6               /* 小画面16px、中画面以上24px */
```



## 出力
- 生成されたソースコードは必ず `src/<画面物理名>/` または `src/components/` 配下に保存すること（命名規則・フォルダ構成に従う）
- フォルダ名は画面物理名（英語）、ファイル名も画面物理名（例：`src/home/home.html`、`src/pos/pos.html`）
- HTML5ファイル（例: home.html, pos.html）
- JavaScript ES6+ファイル（例: home.js, pos.js, utils.js）
- Tailwind CSSクラスを用いたスタイル（CDN経由で読み込み）
- 必要に応じてassets/やcomponents/配下の部品

## チェックリスト（必須）
- 生成されたすべてのコードはエラーがないこと（構文・実行時エラーゼロ）
- 生成されたHTMLのレイアウト・UI配置が画面概要の「画面イメージ」と完全に一致していること
- 設計書（画面一覧・画面概要・画面定義書）に記載された画面・部品の数・要素がすべて揃っていること
- UI/UX・イベント・バリデーション・権限制御など、設計書の要件をすべて満たしていること
- 命名規則・フォルダ構成・ファイル配置が正しいこと
- コードの可読性・保守性・再利用性が高いこと

## 注意点
- 設計書の内容を忠実に反映し、情報不足は「情報不足」と明記
- 画面概要の「画面イメージ」のレイアウト・UI配置を正確に再現すること
- 命名規則・フォルダ構成を厳守
- すべてのロジックはクライアントサイドで完結
- UI/UX・レスポンシブ・アクセシビリティを重視
- テンプレートやサンプルデータは設計書に従う
