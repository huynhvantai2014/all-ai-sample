# 画面定義書一括作成・管理手順

本手順書は、画面一覧（docs/03_画面系/画面一覧.md）に基づき、すべての画面の画面定義書（docs/03_画面系/<画面論理名>/<画面論理名>_画面定義書.md）を一括で作成・管理するための標準手順です。

## 目的
- 画面一覧に記載された全画面について、設計・業務・機能・UI/UX・エラー・連携・チェックリスト等を網羅した画面定義書を漏れなく作成する。
- 画面定義書は必ず docs\01_templates\画面定義書_Template.md のテンプレートに従い、項目・フォーマット・命名規則（docs\00_intructions\00_naming_and_structure_reference.md）を厳守する。

## インプット
- docs/03_画面系/画面一覧.md（画面ID・論理名・物理名・種別・URL・機能概要・備考）
- docs/01_templates/画面定義書_Template.md（画面定義書テンプレート）
- docs/00_intructions/00_naming_and_structure_reference.md（命名規則・フォルダ構成）
- 必要に応じて user story, 画面概要, 設計資料

## 実行手順
1. docs/03_画面系/画面一覧.md を参照し、全画面の論理名・ID・物理名・機能等を抽出する。
2. 画面ごとに docs/01_templates/画面定義書_Template.md の項目・フォーマットに従い、画面定義書を新規作成する。
   - ファイル名は <画面論理名>_画面定義書.md、フォルダ名は画面論理名（日本語、screen等サフィックス禁止）
   - 命名規則・フォルダ構成は docs/00_intructions/00_naming_and_structure_reference.md に厳守
3. 各画面定義書には以下を必ず記載：
   - 画面ID・論理名・物理名
   - 業務目的・業務価値
   - 主な機能・業務フロー
   - 画面レイアウト・UI/UX観点
   - エラー・例外・メッセージ
   - 他画面との連携・遷移
   - 運用・セキュリティ・特記事項
   - チェックリスト（設計・運用・テスト観点）
   - 必要に応じてAIは最大30%まで独自の創造的提案（改善・追加・最適化）を行ってよい
   - 不足情報は「情報不足」と明記し、参照元user story・設計資料を記載
4. 画面一覧の画面数と画面定義書のファイル数・フォルダ数が一致することを必ず確認
5. 画面定義書は日本語で記載し、docs/03_画面系/<画面論理名>/配下に保存

## 出力
- docs/03_画面系/<画面論理名>/<画面論理名>_画面定義書.md（全画面分）
- 画面定義書は docs\01_templates\画面定義書_Template.md のテンプレートに準拠
- 命名規則・フォルダ構成は docs\00_intructions\00_naming_and_structure_reference.md に厳守
- 画面一覧と画面定義書の数・命名が一致

## 注意点
- テンプレート・命名規則・フォーマットを厳守すること
- 不足情報は「情報不足」と明記し、参照元user story・設計資料を記載すること
- 画面定義書は後で理解しやすく、開発・テスト・運用チームに分かりやすいように作成すること
