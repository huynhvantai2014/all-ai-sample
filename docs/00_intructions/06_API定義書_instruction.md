
# API定義書作成手順

本手順書は「00_naming_and_structure_reference.md」の命名規則・フォルダ構成を必ず参照してください。
API定義書は各画面ごとに保存し、画面フォルダ（例: docs/03_画面系/<画面論理名>/）配下に <API論理名>_API定義書.md というファイル名で管理してください。

## 目的
POSシステムのAPI定義書（API定義書）を作成し、各APIに詳細で一貫性のある、検索しやすく発展性のある、IPAテンプレートに従って定義する手順をガイドします。

## ロール
POSプロジェクトの業務分析、システム設計、API設計の専門家として、各APIの定義書を作成・レビューする責任があります。

## インプット
- 各画面・業務で使用しているAPIのリスト
- API定義書テンプレート: docs/templates/API定義書_Template.md
- 関連する業務、メッセージ、エラー情報

## 作成手順
1. 各業務・画面定義書・システム設計から使用APIのリストを総合する
2. 各APIについてテンプレート（docs/templates/API定義書_Template.md）に従いAPI定義書を作成
   - API ID、名前、エンドポイント、メソッド（GET/POST/PUT/DELETE...）を明記
   - APIの機能概要
   - 入力パラメータ詳細（名前、種類、必須、データ型、長さ、意味解説）
   - 出力パラメータ詳細（フィールド名、データ型、長さ、意味解説）
   - エラーコード、メッセージ、発生条件
   - 業務フロー（入力検証、相関/DBチェック、論理処理、結果返却）
   - 補足コメント、特殊ケース
3. API定義書が使用画面・業務に明確にリンクされていることを確認
4. APIが未定義の場合は追加提案し、詳細説明を記載
5. メッセージ・エラー情報と照合し、返却情報の一貫性を確認

## アウトプット
IPAテンプレートに従い、詳細・一貫性・検索性・発展性のあるAPI定義書が、各画面フォルダ（docs/03_画面系/<画面論理名>/）配下に <API論理名>_API定義書.md というファイル名で保存されていること
チェックリストまたは確認すべき問題リストを作成

## 注意点
- docs/templates/API定義書_Template.md を正確に使用すること
- すべての情報は業務・画面定義書・メッセージ・エラー情報と照合して確認すること
- 情報不足の場合は明記し、開発チームに補充依頼すること
- API定義書はバックエンド・フロントエンド・テストチームが理解しやすいように作成すること
