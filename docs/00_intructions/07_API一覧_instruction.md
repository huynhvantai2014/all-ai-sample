# API一覧作成手順

- 本手順書は「00_naming_and_structure_reference.md」の命名規則・フォルダ構成を必ず参照してください。

## ロール
業務分析・システム設計・API設計の専門家として、全体API一覧の集約・管理・更新を担当する。

## 目的
POSシステムの各画面で使用されるAPI一覧を集約・管理し、検索・テスト・開発の効率化を図る。

## インプット
 各画面定義書（docs/03_画面系/<画面論理名>/<画面論理名>_画面定義書.md）に記載されたAPI一覧
 API定義書：docs/03_画面系/<画面論理名>/<API名>_定義書.md
 API一覧テンプレート：docs/templates/API一覧_Template.md

## 手順
1. 各画面定義書を確認し、画面ごとに使用APIを集約
2. テンプレート（docs/templates/API一覧_Template.md）に従いAPI一覧ファイルを新規作成
   - API名、エンドポイント、メソッド、利用画面、主な機能を明記
   - 業務グループまたは画面ごとにAPIを分類
   - 詳細API定義書へのリンクを添付
   - 要確認事項や特記事項を記載
3. 業務変更やAPI追加時はAPI一覧を必ず更新
4. 業務資料・画面定義書・API定義書と照合し、網羅性・一貫性を担保

## アウトプット
- docs/04_機能系/API一覧.md にAPI一覧Markdownファイルを作成
- API一覧は網羅的・明確・検索しやすいこと
- チェックリストや要確認事項も添付

## 注意事項
- docs/templates/API一覧_Template.md のテンプレートを必ず使用
- 業務資料・画面定義書・API定義書と常に照合
- 情報不足時は明確に注記し、開発チームへ連絡
- API一覧はバックエンド・フロントエンド・テスト担当全員が理解できる内容とする

## 命名規則・フォルダ構成
- 画面フォルダ名は「00_naming_and_structure_reference.md」の規則に従うこと
- 画面フォルダ名は機能を正確に反映し、分かりやすく・覚えやすく・検索しやすいこと
- 画面フォルダ名は日本語（ひらがな・カタカナ・漢字・ラテン文字）で記載
- 画面コード（SRC0001等）はフォルダ・ファイル名に使用しない

## API一覧作成手順（まとめ）
1. docs/screens/API一覧.md にMarkdownファイルを作成
2. 内容は日本語で記載
3. docs/templates/API一覧_Template.md の構成・項目に従う
4. 画面一覧に従い画面フォルダを作成（命名規則は「00_naming_and_structure_reference.md」参照）
5. 各画面定義書からAPIを集約
6. テンプレートに従いAPI一覧を作成
   - API名、エンドポイント、メソッド、利用画面、主な機能を明記
   - 業務グループまたは画面ごとにAPIを分類
   - 詳細API定義書へのリンクを添付
   - 要確認事項や特記事項を記載
7. 業務変更やAPI追加時はAPI一覧を必ず更新
8. 業務資料・画面定義書・API定義書と照合し、網羅性・一貫性を担保
