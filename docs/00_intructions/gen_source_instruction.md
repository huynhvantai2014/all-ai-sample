# ソース生成手順

- ドキュメント命名規則・フォルダ構成は「00_naming_and_structure_reference.md」を参照。

## 1. 出力ファイルの準備
- 出力ファイルのフォルダ構成は「00_naming_and_structure_reference.md」を参照。
- 出力ファイルの命名は「screen_name.tsx」で、`screen_name`は画面名（例：home、login、checkout）。
- 出力ファイルの内容は、画面概要、画面定義書、API定義書、モック画面のHTML/CSS/画像ファイルの4つで構成。
- 出力ファイルのフォルダ構成は、`docs/mock/`または`docs/ui_mock/`にモック画面のファイルが保存されている。
- モック画面のファイル名は、画面名（例：login.html、checkout.html、home.html）で、`SRCxxxx`ではなく、画面名で保存されている。

## 2. ソース生成の手順
1. 画面概要（Screen Overview）を読む
   - 画面の業務、フロー、UI/UXの概要、主なロジックを記述
   - 画面のモック画面の内容と一致しているか確認
2. 画面定義書（Screen Definition）を読む
   - 画面のレイアウト、フィールド、イベント、ロジックの詳細を記述
   - ロジックの検証（例：ログインチェック、リダイレクト）
3. API定義書（API Definition）を読む
   - APIのエンドポイント、リクエスト/レスポンス、画面との連携を記述
   - APIのモック画面の内容と一致しているか確認
4. モック画面（HTML/CSS/画像）を読む
   - モック画面の内容と実際の画面の一致を確認
   - モック画面のファイル名と出力ファイルの命名の一致を確認
5. ソースコードの生成
   - Component UI（React/Next.js）：レイアウト、フィールド、イベント、スタイルがモック画面と一致
   - API handler（Next.js API route）：エンドポイント、ロジックの詳細
      ※注意：API handler（Next.js API route）は必ず本物のMongoDB（src/lib/mongodb.ts経由）に接続すること。モックデータは使用しない。
   - Model/data structure（TypeScript、Mongoose...）：データ構造
   - File構成、変数環境の設定

## 3. 出力ファイルの構成
- `src/app/[screen]/page.tsx`：画面のUIコンポーネント
- `src/app/api/[resource]/route.ts`：画面または業務のAPIハンドラ
- `src/models/[Model].ts`：Model/data structure
- `src/lib/[lib].ts`：各ユーティリティ関数、接続
