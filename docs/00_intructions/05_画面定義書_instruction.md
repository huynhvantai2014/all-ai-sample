# 画面定義書作成手順

- 本手順書は「00_naming_and_structure_reference.md」の命名規則・フォルダ構成を必ず参照してください。

## 目的

POSシステムのAPI定義書（API定義書）の作成手順を定義し、各画面で使用されるAPIが明確に定義されて、一貫性があり、検索しやすく、発展性があることを保証します。

## 動機

API定義書は、APIの仕様を明確に定義し、開発チームの理解を助け、品質の向上に貢献します。

## 責任者

あなたはPOSプロジェクトの業務分析、システム設計、API設計の専門家であり、各APIの定義書を作成・レビューする責任があります。

## 入力

- 各画面で使用されるAPIのリスト（docs/画面定義書/<画面ID>_定義書.md）
- API定義書テンプレート：docs/templates/API定義書_Template.md
- 関連する業務、メッセージ、エラー情報

## 実行手順

1. 各画面定義書を読むことで、画面ごとのAPI使用リストを作成します。
2. 各APIに対して、テンプレートに従ってAPI定義書を作成します（docs/templates/API定義書_Template.md）:
   - API ID、名前、エンドポイント、メソッド（GET/POST/PUT/DELETE...）を明確に記載します。
   - APIの業務機能の説明
   - 入力パラメータ（input）、データ型、必須、意味の説明
   - 出力パラメータ（output）、データ型、意味の説明
   - エラークод、メッセージ、発生条件
   - ネットワーク処理の流れ（validate、処理、結果）
   - 記録すべきポイント、特殊なケース
3. API定義書が使用される画面に明確にリンクされていることを保証します。
4. APIがまだ存在しない場合は、追加APIの提案と詳細な説明をします。
5. メッセージ、エラー情報と照らし合わせて、返却情報が一貫していることを確認します。

## 出力

+ 各画面ごとに「画面定義書」を作成します（docs/画面系/<画面論理名>/画面定義書.md）。
+ 画面定義書は必ず「docs/templates/API定義書_Template.md」のテンプレートに従って作成してください。
+ 画面定義書には、画面で使用されるAPIの定義（API ID、名前、エンドポイント、メソッド、パラメータ、エラー情報等）をすべて記載します。
+ 各APIが明確に定義されて、一貫性があり、検索しやすく、画面使用に合わせて保存されていることを保証します。
+ チェックリストまたは確認すべき問題のリスト

## 注意点

- 正確にテンプレートdocs/templates/API定義書_Template.mdを使用してください。
- 画面定義書、業務、メッセージ、エラー情報と照らし合わせて、実際の内容に一致していることを確認してください。
- 未定義の情報がある場合は、明確に記載して発展チームに補充してください。
- API定義書が後で理解しやすく、バックエンド、フロントエンド、テストチームに理解しやすいように作成してください。
