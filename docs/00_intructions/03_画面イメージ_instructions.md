
# 画面イメージ作成・レビュー統合手順

本手順書は「00_naming_and_structure_reference.md」の命名規則・フォルダ構成を必ず参照してください。

## 目的
各画面の画面概要を分析し、日本の設計基準に基づいた最新UI/UXの画面イメージ資料を作成し、さらに業務要件・UI/UX品質・統一性をレビュー・検証する。

## ロール
日本のUX/UI設計専門家（経験20年）。画面概要をもとに、現代的なUI/UX（Material Design, Fluent UI等）で画面イメージを設計・レビュー。
UI/UX設計の品質を日本語の基準で検証・評価する担当者として、業務要件・画面概要・UI/UX統一性を厳密にチェックする。

## インプット
 画面概要資料：各画面ごとに docs/03_画面系/<画面論理名>/画面概要.md に個別ファイルを作成・管理すること

## 作成・検証手順
1. docs/画面概要配下の各ファイルを分析
2. docs/画面一覧.mdの画面リストに従い、各画面の画面イメージ資料（Markdown）を作成
3. 画面イメージ資料はテンプレート（docs/templates/画面イメージ_Template.md）の構成・項目に従う
4. docs/03_画面系/00_画面イメージに各画面のモックHTML/CSSファイルを作成
5. モックファイルは現代的UI/UX（Material Design, Fluent等）を反映し、日本伝統的UIに依存しない
6. HTML/CSSは可読性・テスト容易性を重視し、ブラウザで直接表示可能なサンプルとする。レイアウトは常に画面全体（width/height 100%）にフィットし、余白や空白スペースが残らないように設計すること（特にスマホ・タブレット表示時）。
7. Mock作成・更新時は docs/03_画面系/00_画面イメージ/index.html にリンクを追加し、全Mockを一覧できるようにする
8. 各画面のモックHTML/CSSファイルを検証し、業務要件・画面概要の記述が一致しているか、色・レイアウト・UI/UXが統一されているかを確認
9. 不足情報や不一致がある場合は「レイアウト情報不足」と明記し、修正点・提案をリストアップ。特に画面概要に記載が不十分な場合は、テスト・レビュー容易性のため、画面上にできる限り多くのボタンや操作UIを創造的に追加すること（例：複数の操作ボタン、アイコン、メニュー、ダイアログ等）。

### 画面イメージ資料の記載内容
- レイアウト全体：ヘッダー、メイン、サイドバー、フッター等の構成を現代的UI/UXで記載
- UI部品一覧：フォーム、テーブル、ボタン、入力欄、カード、モーダル等をMaterial Design/Fluent UI等のスタイルで記載（border-radius, shadow, 明るい色、hover/focus効果、サンセリフフォント、最新アイコン等）
- 不足情報は「レイアウト情報不足」として明記

### 検証観点・チェックリスト
1. モックが画面概要の記述に完全に一致しているか
2. 色、フォント、スタイル、効果が各画面間で統一されているか
3. レイアウト全体（ヘッダー、メイン、サイドバー、フッター）が統一されているか
4. UI要素（フォーム、テーブル、ボタン、INPUT、カード、モーダル等）が現代的なスタイル（border-radius、shadow、hover/focus効果、サンセリフフォント、最新アイコン）か
5. 画面概要とモックに成り立たない要素がある場合、「レイアウト情報不足」として明記
6. モックがブラウザで直接開けるように、検証しやすく、読めるようにする

## アウトプット
+ docs/03_画面系/00_画面イメージ/<画面論理名>.html および <画面論理名>.css のMockファイルを作成
+ 内容は日本語で記載
+ テンプレート（docs/templates/画面イメージ_Template.md）の構成・項目に従う
+ 画面一覧に従い画面フォルダを作成（命名規則は「00_naming_and_structure_reference.md」参照）
+ Mockファイル数は画面一覧の画面数と一致させる
+ すべての画面イメージは共通CSS（docs/03_画面系/00_画面イメージ/base.css）を必ず参照し、色・レイアウト・フォント・UI部品のスタイルを統一すること
+ base.cssにはメインカラー、サブカラー、フォント、ボタン・入力欄・カード・モーダル等の共通レイアウト・サイズ・shadow・radius・hover/focus効果を定義する
+ 検証レポート：各画面の達成点/未達成点、欠落要素、UI/UX統一に必要な修正点をリストアップ
+ UI/UX開発チームのチェックリストとして利用可能な形式でまとめる
