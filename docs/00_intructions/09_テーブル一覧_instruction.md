# テーブル一覧作成手順

- 本手順書は「00_naming_and_structure_reference.md」の命名規則・フォルダ構成を必ず参照してください。

## 目的
POSシステムで使用するテーブル一覧を集約・管理し、検索・テスト・開発の効率化を図る。

## ロール
業務分析・システム設計・データベース設計の専門家として、全体テーブル一覧の集約・管理・更新を担当する。

## インプット
- 業務要件定義書、システム分析資料、ERD、データ設計書、システム要件
- テーブル一覧テンプレート：docs/templates/テーブル一覧_Template.md
- テーブル一覧作成後、各テーブル定義書（docs/テーブル定義書/<テーブル名>_定義書.md）を作成

## 手順
1. 必要なテーブルを洗い出し、docs/テーブル一覧.mdに一覧化（テンプレート：docs/templates/テーブル一覧_Template.md）
2. テーブル一覧完成後、各テーブル定義書（docs/templates/テーブル定義書_Template.md）を作成
3. 新規テーブル発生時はテーブル一覧を先に更新し、定義書を追加
4. 業務変更やテーブル追加時はテーブル一覧を必ず更新
5. 業務資料と照合し、網羅性・一貫性を担保

> 標準手順：必ずテーブル一覧を先に作成・更新し、定義書はその後作成

## アウトプット
- docs/テーブル一覧.md にテーブル一覧Markdownファイルを作成
- テーブル一覧は網羅的・明確・検索しやすいこと
- チェックリストや要確認事項も添付

## 注意事項
- docs/templates/テーブル一覧_Template.md のテンプレートを必ず使用
- 業務資料・テーブル定義書と常に照合
- 情報不足時は明確に注記し、開発チームへ連絡
- テーブル一覧はバックエンド・フロントエンド・テスト担当全員が理解できる内容とする

## 命名規則・フォルダ構成
- 画面フォルダ名は「00_naming_and_structure_reference.md」の規則に従うこと
- 画面フォルダ名は機能を正確に反映し、分かりやすく・覚えやすく・検索しやすいこと
- 画面フォルダ名は日本語（ひらがな・カタカナ・漢字・ラテン文字）で記載
- 画面コード（SRC0001等）はフォルダ・ファイル名に使用しない
