# テーブル定義書作成手順

- 本手順書は「00_naming_and_structure_reference.md」の命名規則・フォルダ構成を必ず参照してください。

## 目的

POSシステムのテーブル定義書（テーブル定義書）を作成し、各テーブルに詳細で一貫性のある、簡単に検索・発展できる資料を確保します。

## 動機

POSシステムの専門家として、各テーブルに詳細で一貫性のある、簡単に検索・発展できる資料を確保します。

## 入力

- 各画面・API・業務で使用しているテーブルのリスト
- テーブル定義書テンプレート: docs/templates/テーブル定義書_Template.md
- 関連する業務、メッセージ、エラー資料

## 実施手順

1. 各業務・画面・APIの定義書から使用しているテーブルのリストを合計します。
2. 各テーブルに、テンプレート（docs/templates/テーブル定義書_Template.md）に従って、テーブル定義書を作成します。
   - テーブル名、物理名、業務の説明
   - 各カラムの詳細定義：論理名、物理名、データ型、長さ、必須、主キー/副キー、デフォルト値、意味の説明
   - インデックス、制約、他のテーブルとの関係（FK、PK、UNIQUE...）
   - 業務にかかわる操作（insert、update、delete、select）
   - 説明が必要な点、特殊なケース
3. テーブル定義書が、画面・API・業務で使用されているかどうかを明確にします。
4. まだ存在しないテーブルの場合は、追加でテーブルを作成し、詳細な説明をテンプレートに従って記述します。
5. メッセージ、エラー資料と照らし合わせて、データの整合性を確認します。

## 出力

- 各テーブルに、docs/テーブル定義書/<テーブル名>_定義書.md または docs/テーブル定義書/<物理名>_定義書.md にファイルを作成
- 各テーブルに詳細で一貫性のある、簡単に検索・発展できる資料が存在
- チェックリストまたは確認が必要な問題のリスト

## 注意点

- テンプレート docs/templates/テーブル定義書_Template.md に従って作成
- 各資料の元データ：業務、画面定義書、API定義書、メッセージ、エラー
- 情報が不足している場合は、明確に記述して発展チームに追加で補充
- テーブル定義書は、バックエンド、フロントエンド、テストチームに理解しやすいように作成

- 参考：「00_naming_and_structure_reference.md」の命名規則・フォルダ構成
