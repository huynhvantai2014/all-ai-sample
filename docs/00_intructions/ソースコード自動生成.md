# LLMへの設計書ベースのNext.jsフルスタックソースコード・API生成指示

## 概要
この指示書は、LLM（大規模言語モデル）に対して、設計ドキュメント（docs\03_画面系配下の画面定義書・画面概要）をもとに、Next.jsフルスタック（App Router, API Route, React, Material UI, MongoDB）で実際に動作する画面のソースコードとAPI（バックエンド処理）を自動生成させるためのガイドです。

## 指示内容
1. **設計書参照**
	- docs\03_画面系配下の各画面フォルダにある「画面定義書.md」「画面概要.md」を必ず参照すること。
	- 項目定義、権限定義、画面遷移、アクション一覧、API定義などの内容を正確に反映する。

2. **ソースコード生成**
	- 画面定義書・画面概要の内容に従い、フロントエンド・バックエンド一体型のNext.js（App Router, API Route, React, Material UI, MongoDB）でソースコードを生成する。
	- UI項目、バリデーション、イベント、画面遷移、権限制御を設計書通りに実装する。
	- 必ずNext.js fullstack構成（App Router, API Route, React, MUI, MongoDB）を使用すること。

3. **API実装**
	- 画面定義書の「アクション一覧」「API定義」に記載されたAPIを、Next.jsのAPI Routeとして実装する。
	- 入力・出力・エラー・処理フローを設計書通りに反映する。
	- MongoDB（mongoose）を使ったデータ操作を推奨。

4. **命名規則・構成**
	- docs\03_画面系のフォルダ・ファイル命名規則を厳守する。
	- 画面論理名・物理名・API名は設計書通り。

5. **補足事項**
	- 設計書に情報不足がある場合は「情報不足」と明記し、仮実装またはコメントで対応する。
	- UI/UX観点や権限制御も設計書の記載を優先する。

## 例
- docs\03_画面系/人材一覧・検索画面/人材一覧・検索画面_画面定義書.md の「項目定義」「アクション一覧」「API定義」をもとに、Next.js（App Router, React, MUI）画面とAPI Route（MongoDB操作）を生成する。
- docs\03_画面系/新規社員追加画面/新規社員追加画面_画面定義書.md の「画面遷移」「権限定義」をもとに、画面遷移・権限チェックをNext.jsで実装する。

---
この指示書をLLMに渡すことで、設計書ベースの高品質なNext.js fullstackソースコード・APIを自動生成できます。
